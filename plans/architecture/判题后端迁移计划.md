# 判题逻辑从前端移到后端（批量交卷方案）

## 1. 背景

当前 quiz.html 逐题提交、前端 substring 判定，后端盲写。判题时机不合理。

## 2. 目标

前端只采集答案不判定，整张卷子做完点"交卷"，后端 JudgeGraph 批量判定，一次性返回全部结果。

## 3. 流程变化

```
之前：做题 → 提交 → 前端判 → 显示反馈 → 下一题（逐题）
之后：做题 → 暂存 → 下一题 → ... → 做完 → "交卷" → 后端批量判 → showResult()
```

## 4. JudgeGraph 结构

```
POST /api/submit-exam { student_id, answers: [...] }
  → server.py：
      ① 修复字段映射：source → section_id，student_id 下沉到每条 answer
      ② 构建 JudgeState，graph.invoke()
  → judge_all 节点：
      初始化每条 is_correct=False, reason="未判定"
      遍历：choice → 字母匹配（处理前标号）
            fill_blank/code_fill → 全角/半角归一化 + 精确匹配
            short_answer/comprehensive → 收集到 LLM 队列
      并发 LLM：asyncio.gather + Semaphore(5) + 30s timeout
      → 正则解析 LLM 响应
      → 异常/超时：降级精确匹配
  → 批量写入：record_attempts_batch(records)，事务保护，IMPORT OR IGNORE 去重
  → 返回 { ok, results: [{ is_correct, reason, method, ... }] }
```

```python
from typing import TypedDict, Optional

class JudgeState(TypedDict):
    student_id: str
    answers: list[dict]     # 输入+输出，前端原样字段：
                            #   question_type, student_answer, correct_answer,
                            #   stem, explanation, section_id, topic, difficulty,
                            #   duration_sec, confidence
                            # judge_all 填充：
                            #   is_correct, reason, method
    llm_client: Optional[object]
```

## 7. 新建 `exam/graph/judge_graph.py`

```python
"""批量判题图。JudgeGraph 类，judge_all 单节点，asyncio.gather 并发 LLM。

用法：
    jg = JudgeGraph(llm_client)
    result = jg.invoke({
        "student_id": "S001",
        "answers": [{"question_type":"choice",...}, ...],
    })
"""
import asyncio
import logging
import re
from langgraph.graph import StateGraph, END
from exam.agents.utils.agent_states import JudgeState

logger = logging.getLogger(__name__)

LLM_TIMEOUT = 30        # 单次 LLM 调用超时秒数
LLM_CONCURRENCY = 5     # LLM 并发上限（Semaphore）


class JudgeGraph:
    def __init__(self, llm_client=None):
        self.llm_client = llm_client
        self.graph = self._build()

    def invoke(self, state: dict) -> dict:
        return self.graph.invoke(state)

    def _build(self):
        graph = StateGraph(JudgeState)
        graph.add_node("judge_all", self._judge_all)
        graph.set_entry_point("judge_all")
        graph.add_edge("judge_all", END)
        return graph.compile()

    # ── 节点 ──

    def _judge_all(self, state: JudgeState) -> dict:
        answers = state["answers"]
        llm_tasks = []

        for i, ans in enumerate(answers):
            ans.setdefault("is_correct", False)
            ans.setdefault("reason", "未判定")

            given = ans.get("student_answer", "").strip()
            correct = ans.get("correct_answer", "").strip()

            if not given:
                ans["is_correct"] = False
                ans["reason"] = "未作答"
                ans["method"] = "rule"
                continue

            qtype = ans.get("question_type", "")

            if qtype == "choice":
                ok = _strip_label(given).upper() == _strip_label(correct).upper()
                ans["is_correct"] = ok
                ans["reason"] = "选项匹配" if ok else f"正确答案为 {correct}"
                ans["method"] = "rule"

            elif qtype in ("short_answer", "comprehensive"):
                llm_tasks.append((i, given, correct,
                                  ans.get("stem", ""),
                                  ans.get("explanation", "")))

            else:  # fill_blank / code_fill
                ok = _normalize(given) == _normalize(correct)
                ans["is_correct"] = ok
                ans["reason"] = "答案匹配" if ok else f"正确答案为 {correct}"
                ans["method"] = "rule"

        if llm_tasks:
            logger.info("judge_all: %d 道规则判定，%d 道 LLM 判定",
                        len(answers) - len(llm_tasks), len(llm_tasks))
            _run_llm_batch(answers, llm_tasks, self.llm_client)

        return {"answers": answers}


# ── LLM 并发 ──

def _run_llm_batch(answers, tasks, llm_client):
    if llm_client is None:
        for i, given, correct, _, _ in tasks:
            answers[i]["is_correct"] = (given == correct)
            answers[i]["reason"] = "降级（LLM 未配置）"
            answers[i]["method"] = "fallback"
        return

    async def _gather():
        sem = asyncio.Semaphore(LLM_CONCURRENCY)
        async def _one(i, given, correct, stem, expl):
            async with sem:
                try:
                    ok, reason = await asyncio.wait_for(
                        asyncio.get_event_loop().run_in_executor(
                            None, lambda: _call_llm(llm_client, stem, correct, given, expl)
                        ),
                        timeout=LLM_TIMEOUT,
                    )
                    answers[i]["is_correct"] = ok
                    answers[i]["reason"] = reason
                    answers[i]["method"] = "llm"
                except asyncio.TimeoutError:
                    answers[i]["is_correct"] = (given == correct)
                    answers[i]["reason"] = "降级（LLM 超时）"
                    answers[i]["method"] = "fallback"
                    logger.warning("LLM 超时: idx=%d", i)
                except Exception as e:
                    answers[i]["is_correct"] = (given == correct)
                    answers[i]["reason"] = "降级（LLM 异常）"
                    answers[i]["method"] = "fallback"
                    logger.warning("LLM 异常: idx=%d, err=%s", i, e)
        await asyncio.gather(*[_one(*t) for t in tasks])

    try:
        loop = asyncio.get_event_loop()
        if loop.is_running():
            import threading
            t = threading.Thread(target=asyncio.run, args=(_gather(),))
            t.start()
            t.join()
        else:
            asyncio.run(_gather())
    except RuntimeError:
        # 最终兜底：降级为精确匹配
        for i, given, correct, _, _ in tasks:
            answers[i]["is_correct"] = (given == correct)
            answers[i]["reason"] = "降级（事件循环异常）"
            answers[i]["method"] = "fallback"
        logger.exception("LLM 批量判定事件循环异常")


def _call_llm(llm_client, stem, correct, given, expl):
    from langchain_core.messages import HumanMessage
    prompt = (
        f"题目：{stem}\n"
        f"参考答案：{correct}\n"
        f"学生答案：{given}\n"
        f"题解（供参考）：{expl}\n\n"
        f"学生答案在语义上是否正确？只回复 true 或 false，然后一句话理由。"
    )
    response = llm_client.invoke([HumanMessage(content=prompt)])
    text = response.content.strip()
    match = re.search(r'\b(true|false)\b', text, re.IGNORECASE)
    ok = match and match.group(1).lower() == "true"
    reason = re.sub(r'\btrue\b|\bfalse\b', '', text, flags=re.IGNORECASE).strip(" ,.，。:")
    return ok, reason or "LLM 判定"


# ── 文本工具 ──

def _normalize(s: str) -> str:
    """空格压缩、全角转半角。"""
    s = s.strip().replace('　', ' ').replace('\xa0', ' ')
    s = ' '.join(s.split())
    result = []
    for c in s:
        if '０' <= c <= '９': result.append(chr(ord(c) - 0xFEE0))
        elif 'Ａ' <= c <= 'Ｚ': result.append(chr(ord(c) - 0xFEE0))
        elif 'ａ' <= c <= 'ｚ': result.append(chr(ord(c) - 0xFEE0))
        else: result.append(c)
    return ''.join(result)


def _strip_label(s: str) -> str:
    """去掉选项前标号 'A.' / 'A、' → 'A'。"""
    return re.sub(r'^[A-Da-d][.、．\s]+', '', s).strip()
```

### 8.2 新增批量写入函数

```python
def record_attempts_batch(db_path: str, records: list[dict]):
    """批量写入 attempts，事务保护。student_id 从每条 record 中取。"""
    db = sqlite3.connect(db_path)
    try:
        with db:
            for r in records:
                db.execute(
                    """INSERT OR IGNORE INTO attempts
                       (student_id, section_id, topic, question_type, difficulty,
                        stem, student_answer, correct_answer, explanation,
                        is_correct, duration_sec, confidence, reason, method)
                       VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)""",
                    (
                        r.get("student_id", ""),
                        r.get("section_id", ""),
                        r.get("topic", ""),
                        r.get("question_type", ""),
                        r.get("difficulty", ""),
                        r.get("stem", ""),
                        r.get("student_answer", ""),
                        r.get("correct_answer", ""),
                        r.get("explanation", ""),
                        1 if r.get("is_correct") else 0,
                        r.get("duration_sec", 0),
                        r.get("confidence", 3),
                        r.get("reason", ""),
                        r.get("method", "rule"),
                    ),
                )
    finally:
        db.close()
```

```python
def do_POST(self):
    parsed = urlparse(self.path)
    if parsed.path == "/api/submit-exam":
        length = int(self.headers.get("Content-Length", 0))
        body = self.rfile.read(length) if length else b"{}"
        try:
            data = json.loads(body)
            student_id = data.get("student_id", "")

            # ① 修复字段映射 + 下沉 student_id
            for ans in data.get("answers", []):
                ans["section_id"] = ans.pop("source", ans.get("section_id", ""))
                ans["student_id"] = student_id

            # ② 调图
            state = {"student_id": student_id, "answers": data["answers"]}
            result = JUDGE_GRAPH.invoke(state)

            # ③ 批量写入（事务保护）
            record_attempts_batch(ATTEMPTS_DB, result["answers"])

            # ④ 返回

                "is_correct": a["is_correct"],
                "reason": a["reason"],
                "method": a.get("method", "rule"),
                "correct_answer": a["correct_answer"],
                "explanation": a.get("explanation", ""),
            } for a in result["answers"]]
            self._serve_json({"ok": True, "results": results})

            logger.exception("submit-exam 失败")
```

## 12. 判定规则

| 题型 | 规则 | method |
|------|------|--------|
| `choice` | `_strip_label(given).upper() == _strip_label(correct).upper()` | `rule` |
| `fill_blank` / `code_fill` | `_normalize(given) == _normalize(correct)`（全角→半角、压缩空格） | `rule` |
| `short_answer` / `comprehensive` | LLM 语义判定（含 stem + correct + explanation），正则解析 true/false | `llm` |
| LLM 超时/异常/未配置 | 降级精确匹配 | `fallback` |

## 13. 并发与超时

- LLM 并发上限：`Semaphore(5)`
- 单次 LLM 超时：30s
- 前端 fetch 超时：`AbortController` 30s
- 降级链：LLM 超时 → 精确匹配 → reason 标注降级 → method=`fallback`

## 14. 关键设计修正（红蓝对抗后）

| 修正 | 问题 | 修法 |
|------|------|------|
| 事件循环兼容 | `asyncio.run()` 空吞 RuntimeError | 检测 `loop.is_running()`，用线程或同步 |
| LLM prompt | 缺 stem + explanation 标签错误 | prompt 含 `题目：{stem}`，`explanation` 标"题解" |
| 字段映射 | `source` ≠ `section_id`，`student_id` 未下沉 | server 里 `ans["section_id"] = ans.pop("source", ...)`；`ans["student_id"] = student_id` |
| 事务保护 | 判一半炸了无法回滚 | `record_attempts_batch()` 用 `with db:` 事务 |
| LLM 超时 | 无限等待 | `asyncio.wait_for(timeout=30)` |
| LLM 解析 | `startswith("true")` 脆弱 | 正则 `\b(true|false)\b` |
| 文本框归一 | 全角/空格导致误判 | `_normalize()` 全角→半角、压缩空格 |
| 选项标号 | 学生传 "A." vs "A" | `_strip_label()` |
| 日志 | 无日志 | `logging.getLogger(__name__)` |
| 并发上限 | 30 道简答题同时 LLM | `Semaphore(5)` |
| 风格统一 | judge_graph 函数式 vs exam_graph 类 | `JudgeGraph` 类 |
