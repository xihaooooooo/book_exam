# Student Profile 完整方案

## 1. 背景：系统现状

```
PDF 教材 → 解析入库 → LLM Agent 出题流水线 → 生成试卷
                                    ↑
                              支持 exam / practice / diagnostic 三种模式
```

**practice 模式现状**：读错题库，按章节错误次数加权出题。缺点是只记录错题，只知道"某章节错了 N 次"，无法判断为什么错、该怎么针对性练。

**核心判断**：通用出题能力容易被大模型直接替代。更有独特性的方向是建立学生长期画像，让系统能回答"薄弱在哪 → 为什么薄弱 → 接下来该练什么"。

---

## 2. 目标闭环

```
学生作答
   → ① 记录 attempt（作答事实：答了什么、对错、耗时、信心度）
   → ② 诊断错因（为什么错：概念混淆？推理错误？粗心？）
   → ③ 聚合画像（按章节/考点算掌握等级、主要错因、风险信号）
   → ④ 生成练习计划（弱点优先，根据错因匹配题型和难度）
   → ⑤ 下一轮 practice 按计划出题 → 循环回 ①
```

关键点：闭环的重点不是展示画像，而是让画像**真正影响下一次出题策略**。

---

## 3. 与现有错题库的区别

| | 现有 mistakes.db | 新增 attempts 表 |
|---|---|---|
| 记录范围 | 只记错题 | 每一次作答（对错都记） |
| 耗时 | 无 | 有 → 区分"不熟"和"不会" |
| 信心度 | 无 | 有 → 发现伪掌握（高信心答错） |
| 题型 | 无 | 有 → 判断迁移能力 |
| 能回答的问题 | "错了什么" | "为什么错、下一步练什么" |

## 4. 数据模型

### 4.1 `attempts` — 作答事实

记录每一次作答，作为学生画像的事实来源。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| student_id | TEXT | 学生标识 |
| section_id | TEXT | 章节编号 |
| topic | TEXT | 知识点 |
| question_type | TEXT | choice / fill_blank / short_answer / comprehensive |
| difficulty | TEXT | easy / medium / hard |
| stem | TEXT | 题干 |
| student_answer | TEXT | 学生答案 |
| correct_answer | TEXT | 标准答案 |
| is_correct | INTEGER | 0/1 |
| duration_sec | INTEGER | 作答耗时 |
| confidence | INTEGER | 学生自评把握度 1-5 |
| created_at | TEXT | 时间戳 |

### 4.2 `attempt_error_labels` — 错因标签

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| attempt_id | INTEGER FK | 关联 attempts.id |
| error_type | TEXT | 固定 6 类之一 |
| confidence | REAL | 诊断置信度 |
| source | TEXT | student / teacher / manual / llm |
| evidence | TEXT | 诊断证据 |
| suggestion | TEXT | 改善建议 |
| created_at | TEXT | 时间戳 |

一条 attempt 可以有多个 error label，MVP 阶段取置信度最高的作为主错因。

### 4.3 `student_skill_stats` — 画像聚合

按 (student_id, section_id, topic) 三元组聚合。

| 字段 | 说明 |
|------|------|
| attempt_count | 总作答次数 |
| wrong_count | 错误次数 |
| accuracy | 整体正确率 |
| recent_accuracy | 近期正确率 |
| avg_duration_sec | 平均耗时 |
| avg_confidence | 平均信心 |
| dominant_error_type | 主要错因 |
| streak_wrong | 连续错误次数 |
| mastery_level | unknown / weak / unstable / familiar / mastered |
| needs_review | 是否需要复习 |
| updated_at | 更新时间 |

### 4.4 `practice_plans` — 练习计划

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| student_id | TEXT | |
| focus_sections | TEXT | JSON 数组，如 `["2.3","3.1"]` |
| focus_topics | TEXT | JSON 数组 |
| question_types | TEXT | JSON 数组，如 `["choice","short_answer"]` |
| difficulty | TEXT | easy_to_medium / medium / etc. |
| target_count | INTEGER | 建议题数 |
| reason | TEXT | 推荐原因 |
| created_at | TEXT | |
| used_at | TEXT | 标记是否已被 practice 使用 |

## 5. 错因分类体系（6 类固定）

| 错因 | 含义 | 典型表现 |
|------|------|----------|
| `concept_confusion` | 概念混淆 | 把任务状态切换和调度触发混在一起 |
| `memory_gap` | 记忆缺失 | 记不住定义、公式、关键条件 |
| `reasoning_error` | 推理错误 | 知道概念但推理链走错 |
| `misread_question` | 审题错误 | 漏看"错误的是"、"不包含"等关键词 |
| `careless` | 粗心失误 | 会做但抄错、选错、提交失误 |
| `transfer_failure` | 迁移失败 | 选择题会对，换简答或综合题就不会 |

错因优先级（越靠前越需要系统性训练）：

```
concept_confusion > reasoning_error > transfer_failure
> memory_gap > misread_question > careless
```

---

## 6. 掌握等级计算（5 档）

| 等级 | 判定规则 |
|------|----------|
| `unknown` | 作答次数 < 2 |
| `weak` | recent accuracy < 0.5，或连续错误 ≥ 2 |
| `unstable` | recent accuracy 0.5-0.75，或高信心错误较多 |
| `familiar` | recent accuracy ≥ 0.75，但耗时偏长或信心偏低 |
| `mastered` | recent accuracy ≥ 0.85，平均信心高，最近无连续错误 |

---

## 7. 风险信号识别

| 信号 | 条件 | 含义 |
|------|------|------|
| 高信心错误 | confidence ≥ 4 且答错 | 伪掌握 |
| 慢速正确 | 答对但耗时远超该生同类题平均 | 会但不熟 |
| 快速错误 | 耗时很短且答错 | 可能是粗心或审题问题 |
| 连续错误 | 同一 section/topic 连错 ≥ 2 | 重点复练 |
| 迁移失败 | 基础题对但综合题错，或同知识点换题型后错 | 迁移能力不足 |

---

## 8. 练习推荐策略

根据 `student_skill_stats` 的 `dominant_error_type` 和 `mastery_level` 生成 practice plan：

| 主要错因 | 推荐策略 |
|----------|----------|
| `memory_gap` | easy 难度，choice / fill_blank，先记牢 |
| `concept_confusion` | choice / short_answer，加入辨析题 |
| `reasoning_error` | short_answer / comprehensive，easy → medium 逐步 |
| `misread_question` | 带限制条件、反向问法的选择题 |
| `careless` | 控制难度，加限时和复核提示 |
| `transfer_failure` | 同知识点不同题型或小综合题 |

通用规则：
- 优先选 `mastery_level = weak` 的知识点
- 其次选 `unstable` 且最近练习较早的
- 少量 `familiar` 知识点做保持性复习

## 9. 模块结构

新增目录 `exam/student_profile/`：

| 模块 | 职责 |
|------|------|
| `schemas.py` | 定义 attempts / error_label / skill_stat / practice_plan 数据结构 |
| `storage.py` | 建表、读写（四个表统一管理） |
| `attempt_recorder.py` | 对外提供记录作答的入口 |
| `profile_engine.py` | 规则聚合画像（mastery level、风险信号、错因分布） |
| `recommender.py` | 根据画像生成下一轮 practice plan |
| `diagnosis_agent.py` | LLM 辅助错因诊断（阶段 5） |

对外 CLI：

| 脚本 | 用途 |
|------|------|
| `record_attempt.py` | 命令行录入一次作答 + 错因标签 |
| `show_profile.py` | 查看学生画像 + `--recommend` 生成练习计划 |

---

## 10. 与现有 practice 模式的集成

`generate.py --mode practice --student S001` 的升级逻辑：

```
① 优先读 practice_plans 表，取最新未使用的 plan
     → 有 → 按 plan 的 focus / count / types / difficulty 出题
② 无 → 回退到现有 mistakes.db 的 weak_sections
     → 有 → 按错题分布出题（现有逻辑）
③ 无 → 降级为 exam 模式或提示先做诊断练习
```

不破坏现有行为，完全向后兼容。

## 11. 命令行设计

```bash
# 记录一次作答
python record_attempt.py --student S001 --section 2.3 --topic 任务调度 \
    --type choice --difficulty easy --correct false --duration 80 \
    --confidence 5 --error concept_confusion

# 查看学生画像
python show_profile.py --student S001

# 生成下一轮练习计划
python show_profile.py --student S001 --recommend

# 按画像出题
python generate.py --mode practice --student S001
```

## 14. MVP 验收标准

- [ ] 能记录一次作答（而不只是记录错题）
- [ ] 能标注一次错误的错因
- [ ] 能按学生/章节/考点聚合正确率、近期正确率、连续错误、主要错因
- [ ] 能输出学生薄弱知识点 Top 5、主要错因分布、下一轮练习建议
- [ ] `practice` 模式能优先使用画像推荐，无画像时回退到旧错题库

## 15. 设计决策（已定）

经过全局最优审查后，以下三个决策已确认：

### 15.1 数据源：attempts 为主，mistakes 为回退

- 新增作答**只写** attempts 表
- `get_weak_sections()` 优先查 attempts（`WHERE is_correct=0`），attempts 为空时回退查 mistakes
- 现有 mistakes 表结构和写入代码保留不动，但不新增写入路径
- 等闭环验证通过后再决定是否彻底废弃 mistakes.db

### 15.2 画像聚合：MVP 不做物化表，实时计算

- 不建 `student_skill_stats` 表
- `profile_engine.py` 每次被调用时从 attempts + error_labels 实时聚合
- 单学生数据量（几十到几百条）完全够用，没有同步问题
- 后续数据量大时再引入物化缓存

### 15.3 表定义：统一到 storage.py

- `web/server.py` 删除自己的 `_init_db()` 和 `record_answer()`
- 改为 `from exam.student_profile.storage import init_attempts_db, record_attempt`
- 表结构只在 storage.py 一处定义
